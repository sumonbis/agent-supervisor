Agentic-SE Coach

A review website that turns a whole class’s repositories into a weekly supervision assessment. It scores team projects on the Supervision Skill Model directly from git history, makes the automation-complacency signature visible, and supports fair multi-committer review.

What it measures

Two SSM skills, computed from the repository

Verification and workflow are derivable from git without running code or a model. The remaining four skills are reserved for an optional instructor LLM pass and are shown as pending.

The teaching signal

Automation complacency, made visible

When organized workflow (S6) runs persistently ahead of verification (S3), the team is leaning on the agent faster than it is checking the agent. The weekly trajectory shades that gap so an instructor can see exactly when it opened.

In development

A student VS Code extension

Coach for the student IDE (in progress). A companion VS Code extension lets students push an abstracted supervision snapshot from their own editor into the class review site — computed locally with the same analysis core, carrying only scores and signal counts (no source, file paths, emails, commit hashes, or identities). This closes the loop between the student’s workflow and the instructor’s review, and is part of the project’s ongoing education research. An early prototype exists; the workflow is not yet finalized.
Data & privacy. The public demo is generated from 18 real student team projects studied under IRB approval. The dataset is anonymized at generation time — only scores, counts, and signal flags are emitted (Committer A / Project NN). No source code, file paths, emails, commit SHAs, or real identities are ever published, and the source repositories are not included.
